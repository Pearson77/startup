# MVP архитектура генератора API-тестов на базе LLM-агентов

На схеме показан минимально жизнеспособный вариант системы для генерации и проверки API-тестов на основе спецификации и цепочки LLM-агентов.

Пользователь загружает спецификацию API в систему. Далее данные проходят через набор базовых агентов: сначала выполняется обработка и извлечение структуры спецификации, затем агент расширяет краткое техническое задание в подробное описание, после чего следующий агент формирует последовательность сценариев с указанием эндпоинтов, порядка их выполнения и необходимых данных. На следующем этапе агент преобразует сценарии в JSON-тесты.

Отдельно предусмотрен интерфейс для ручной проверки: пользователь может просматривать список эндпоинтов, менять их порядок на этапе генерации сценариев и редактировать JSON, который будет использован для запуска тестов.

После подготовки тестов контроллер отправляет их на HTTP-клиент, получает ответы от целевого API и передает результаты на дополнительную проверку агентам. Эти агенты сопоставляют фактический ответ с ожидаемым поведением по сценарию, после чего система формирует итоговый отчет о прохождении тестирования.

Схема отражает полный цикл работы MVP: от загрузки спецификации и генерации сценариев до запуска тестов, анализа ответов и просмотра итоговых результатов.
